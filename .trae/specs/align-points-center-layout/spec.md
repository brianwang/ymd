# 积分中心按钮与布局对齐 Spec

## Why
当前「积分中心」页面的任务按钮与内容区存在对齐与节奏不一致的问题，导致观感不够专业，影响可读性与操作效率。

## What Changes
- 统一「任务中心」三行任务的右侧操作区宽度与对齐方式，使按钮在视觉上严格对齐（同一右边界/同一中心线）。
- 统一任务按钮的尺寸规范（高度、内边距、字号）与状态表现（disabled/ghost/primary）。
- 优化顶部余额卡片头部排版：左右信息在同一基线对齐；昵称过长时截断不挤压布局。
- 优化「积分流水」列表右侧分值列：数值右对齐、最小宽度一致，避免跳动。
- **不新增业务逻辑**，仅调整布局结构与样式。

## Impact
- Affected specs: 积分中心 UI 观感一致性、可读性、触达性
- Affected code:
  - ymd-app：`src/pages/points/index.vue`
  - （如需要）共享 UI：`src/components/ui/*`（仅在发现是组件层问题时修改）

## ADDED Requirements

### Requirement: 任务中心按钮对齐
系统 SHALL 在积分中心的任务列表中提供一致的右侧操作区布局，使三行任务按钮在同一垂直对齐线上呈现。

#### Scenario: 不同文案长度仍对齐
- **WHEN**「去签到 / 领取奖励 / 去邀请 / 已完成」等不同长度的按钮文案在同一列表中显示
- **THEN** 右侧按钮的占位宽度一致，按钮内容居中，按钮整体在三行中保持同一对齐基准（同一右边界或同一中心线）
- **AND** 按钮高度与上下留白一致，不出现单行被“撑高/挤压”

### Requirement: 余额卡片头部信息对齐
系统 SHALL 在余额卡片顶部区域保持左右信息在同一视觉基线对齐，并在昵称过长时保持布局稳定。

#### Scenario: 昵称过长
- **WHEN** 用户昵称长度超出可用空间
- **THEN** 昵称以省略号截断，不与左侧「当前积分」标签重叠、不挤压数值区块

### Requirement: 流水分值列稳定对齐
系统 SHALL 将积分流水每条记录的分值列右对齐并保持最小宽度，从而避免滚动时数字列“左右跳动”。

#### Scenario: 正负数混排
- **WHEN** 流水列表同时出现正数（+x）与负数（-x）
- **THEN** 分值列右对齐、宽度稳定，视觉上形成对齐的数字列
