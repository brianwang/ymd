# 社区图文音视频发布与排版重构 Plan

## Summary
重新设计并实现社区发帖（`create.vue`）与媒体展示（`PostMedia.vue`）的排版。将发帖页改为“底部工具栏 + 互斥媒体选择”的现代社交 App 风格；将帖子媒体流重构为“单图大图/多图九宫格 + 视频大屏带播放 icon”的成熟朋友圈/微博排版模式。

## Current State Analysis
- **发帖页 (`create.vue`)**：
  - 现为平铺的三个大号 `ghost` 按钮（图片/视频/音频），挤占了输入区。
  - 目前允许同一篇帖子同时选择并上传图片、视频、音频，逻辑复杂且界面混乱。
  - 已上传的缩略图（104x104）及移除按钮样式偏粗糙，视频和音频块的进度条/封面缺乏精致感。
- **媒体渲染 (`PostMedia.vue`)**：
  - 图片不论数量，均使用固定的 `104x104` 尺寸横向折行排布（没有单图大屏或九宫格自适应逻辑）。
  - 视频封面仅有右下角的黑色半透明时长气泡，缺乏明确的“点击播放”视觉引导（Play Icon）。
  - 列表（Feed）中通过优先级粗暴忽略次要媒体，与“单类型互斥”的社交逻辑脱节。

## Proposed Changes

### 1) 发帖页 (`create.vue`) 交互与 UI 重构
**What & How**
- **底部工具栏**：取消平铺的大按钮，将“图片 / 视频 / 音频”收纳到贴近键盘/底部的 `toolbar`，并使用 `u-icon`（如 `photo`, `play-circle`, `mic`）提升精致感。
- **媒体单选互斥**：
  - 增加逻辑限制：一旦用户选择了某一种媒体（例如选了 1 张图），底部的“视频”和“音频”图标变灰（不可点击），强制只能发一种媒体类型。
  - 用户清空当前媒体后，恢复所有选项的可点击状态。
- **全屏排版与预览区**：
  - `textarea` 占据上方空间，底部显示选中的媒体预览区（采用 3x3 九宫格布局渲染选中图片，或单行大卡片渲染视频/音频）。
  - 移除丑陋的 `104x104` 写死尺寸，图片预览框采用正方形占位（如 31% 宽度）。

### 2) 媒体展示组件 (`PostMedia.vue`) 渲染重构
**What & How**
- **图片九宫格排版自适应**：
  - `1张图`：采用大图模式（最大宽度占容器的 70% 左右，保留原图比例或裁切合适的高宽比）。
  - `2-4张图`：采用 `2x2` 排版（两列，每列约 48%）。
  - `5-9张图`：采用 `3x3` 九宫格排版（三列，每列约 32%）。
- **视频大屏展示增强**：
  - 将视频卡片宽高比优化为 16:9（或满宽）。
  - 在视频封面中央叠加一个醒目的半透明 `Play` 播放图标（例如通过一个圆形背景 + 三角形的纯 CSS/SVG 或 u-icon 实现），让用户一眼看出是视频。
  - 详情页 (`mode='detail'`) 继续渲染 `<video>` 组件播放，但在 Feed 流 (`mode='feed'`) 仅渲染带播放按钮的精美封面。
- **统一互斥渲染逻辑**：
  - 移除原先复杂的 `v-else-if` 回退逻辑，直接根据帖子挂载的媒体类型（仅一种）精确渲染对应的模块。

## Assumptions & Decisions
- **组件库**：项目已安装 `uview-plus`，将使用 `<u-icon>` 作为发帖页底部工具栏的图标。
- **数据结构**：后端 `app/schemas/post.py` 原本支持多媒体数组，重构为单类型互斥后，前端提交的 `media` 数组只会包含一种类型（但接口层面向下兼容，无需修改后端 API）。
- **样式体系**：继承前面刚做完的扁平化与浅色渐变背景规范，阴影极弱，边框清晰。

## Verification Steps
1. **类型与编译**：
   - 运行 `npm run type-check` 和 `npm run build:h5`，确保无新增错误。
2. **人工回归点**：
   - 在 H5/小程序 访问 `/pages/community/create`，确认工具栏沉底、媒体图标清晰；选择一张图片后，视频/音频图标置灰。
   - 删除图片后，视频/音频恢复可用；选择视频并提交，能正常发布。
   - 访问社区首页 `/pages/community/index`，查看 1 张图、4 张图、9 张图的帖子，验证九宫格排版是否正确。
   - 查看含视频的帖子，确认中央有明确的播放图标。
