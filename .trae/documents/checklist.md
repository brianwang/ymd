# 测试与验收清单 (Checklist)

## 一、基础设施验收 (Infrastructure Verification)
- [ ] 1. 确认 `docker-compose up -d` 能够成功启动 PostgreSQL 和 Redis 且容器运行稳定。
- [ ] 2. 确认后端 FastAPI 服务能正常启动，Swagger UI (`http://localhost:8000/docs`) 能够访问。
- [ ] 3. 确认前端 UniApp (Vue 3) 项目能在 HBuilderX 或微信开发者工具中正常编译和运行，无报错。
- [ ] 4. 确认 Alembic 数据库迁移成功，核心表（`users`, `points_ledger`, `events`, `spaces`, `posts` 等）已创建。

## 二、核心业务流程验收 (Core Business Logic)
- [ ] **用户注册登录 (Auth)**：
  - [ ] 新用户进入小程序，点击授权登录，成功获取用户信息并注册。
  - [ ] 登录成功后，前端正确存储 JWT Token 并在后续请求中携带。
  - [ ] 用户能成功编辑个人资料（昵称、头像、职业标签、MBTI）。
- [ ] **裂变引擎与积分体系 (Viral & Points)**：
  - [ ] 用户A能成功生成带有自身ID参数的邀请海报。
  - [ ] 用户B通过扫描A的海报进入小程序并注册，A和B的“情感账本”均增加相应积分。
  - [ ] 用户完成每日签到、首次发帖，积分流水正确记录且余额更新。
  - [ ] 积分兑换/抵扣逻辑（预留接口测试通过）。
- [ ] **活动报名 (Events)**：
  - [ ] 首页能正确展示活动列表，筛选功能（城市、类型）正常工作。
  - [ ] 点击活动进入详情页，展示富文本、时间地点等信息无误。
  - [ ] 用户点击报名，能成功提交报名信息（单人或拼团）。
  - [ ] 报名成功后弹出分享引导（裂变触点）。
- [ ] **社区互动 (Community)**：
  - [ ] 社区Tab能正常分页加载动态列表。
  - [ ] 用户能成功发布包含文字和多张图片的动态。
  - [ ] 用户能对帖子进行点赞，取消点赞。
  - [ ] 用户能在帖子下发表评论，评论列表能正确显示。
- [ ] **共居空间预订 (Co-living Spaces)**：
  - [ ] 共居Tab能正常展示空间列表，筛选（城市、设施）正常工作。
  - [ ] 点击空间进入详情页，轮播图、设施列表、入住用户墙展示正常。
  - [ ] 用户点击预订/咨询，能成功提交包含入住时间的意向表单。
  - [ ] 会员价格与普通价格的显示逻辑（如果有）正确区分。

## 三、性能与体验验收 (Performance & UX)
- [ ] 图片上传时能自动进行压缩（不超过微信限制）。
- [ ] 列表页的下拉刷新、上拉加载更多功能正常。
- [ ] 所有表单提交有防抖处理，避免重复提交。
- [ ] 网络请求失败时有友好的提示（Toast），不出现白屏。
- [ ] 核心页面的加载骨架屏或 Loading 动画正常显示。
- [ ] 分享卡片（分享到微信群或朋友）的标题、描述和封面图符合预期设计。
