# 开发任务拆解 (Tasks)

## 阶段一：项目初始化与基础设施 (Infrastructure Setup)
- [ ] 1.1 创建前后端项目目录 (`ymd-app`, `ymd-server`)，初始化 Git 仓库。
- [ ] 1.2 编写 `docker-compose.yml` 启动 PostgreSQL 15+ 与 Redis（用于缓存和Celery队列）。
- [ ] 1.3 **后端**：配置 FastAPI 基础脚手架，包含 SQLAlchemy、Alembic 迁移工具、Pydantic v2 模型。
- [ ] 1.4 **后端**：配置 JWT 认证机制与依赖注入 (Dependency Injection)。
- [ ] 1.5 **前端**：初始化 UniApp (Vue 3 + TS) 项目，引入 UI 框架（如 uView Plus）。
- [ ] 1.6 **前端**：配置 Pinia 状态管理，封装基于 Promise 的网络请求模块 (axios/luch-request)。

## 阶段二：用户认证与基础资料 (Auth & Profile)
- [ ] 2.1 **后端**：实现微信小程序 `wx.login` 获取 OpenID/UnionID 的接口。
- [ ] 2.2 **后端**：实现用户注册/登录 API，生成并返回 JWT Token。
- [ ] 2.3 **后端**：实现用户基础信息（昵称、头像、职业标签等）的获取与更新 API。
- [ ] 2.4 **前端**：实现微信授权登录逻辑与 Token 存储。
- [ ] 2.5 **前端**：实现“我的”页面基础布局与个人资料编辑页。

## 阶段三：积分体系（情感账本）与裂变引擎 (Viral Engine & Points)
- [ ] 3.1 **后端**：设计 `points_ledger` 积分流水表，实现积分增加/扣减的事务管理。
- [ ] 3.2 **后端**：实现每日签到、发帖奖励、邀请好友奖励等基础积分任务 API。
- [ ] 3.3 **后端**：实现带参数的小程序码生成（用于生成邀请海报），解析邀请人 ID 绑定上下级关系。
- [ ] 3.4 **前端**：实现“情感账本”页面，展示积分余额与流水明细。
- [ ] 3.5 **前端**：实现生成专属邀请海报的画布合成（Canvas）与保存功能。

- [ ] 4.3 **前端**：实现首页活动列表页（含筛选功能）与日历视图。
- [ ] 4.4 **前端**：实现活动详情页，包含富文本展示、报名浮层与分享引导。
- [ ] 4.5 **全栈**：实现报名支付流程（MVP 阶段可先实现积分抵扣或模拟支付）。
