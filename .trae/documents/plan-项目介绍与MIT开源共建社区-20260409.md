## Summary
- 目标：为“游牧岛（Nomad Island）”补齐开源项目对外介绍与最小共建入口，包含：根目录 README（中文为主）+ MIT LICENSE + CONTRIBUTING。
- 成功标准：新访客在 3 分钟内能理解项目是什么、包含哪些模块、如何本地跑起来、如何参与共建；仓库根目录具备 MIT 授权文件且贡献路径清晰可执行。

## Current State Analysis（基于仓库现状）
- 仓库结构：单仓库包含三端
  - ymd-server：FastAPI + SQLAlchemy(Async) + Alembic + PostgreSQL；提供 /api/v1 接口（auth/users/posts/events/points/coliving/admin 等）。
  - ymd-app：UniApp（Vue3）+ Pinia + Vite；Tab：活动/社区/共居/我的。
  - ymd-admin：Vue3 + Vite + Pinia；管理端通过 /api/v1/admin/* 访问 admin API。
- 启动与开发：仓库已有一键启动脚本与说明（scripts/README-dev-start.md、根目录 dev-start.ps1）。
- 对外文档缺口：
  - 根目录缺少 README.md（无法在代码托管平台首页展示项目介绍）。
  - 根目录缺少 LICENSE 文件（当前无法明确 MIT 授权）。
  - 缺少 CONTRIBUTING（参与路径未成文，难以形成稳定共建流程）。

## Assumptions & Decisions（已确认/默认）
- README 语言：中文为主，关键段落可附英文一句话摘要（用户已选择“中文为主”）。
- 共建入口：默认使用 GitHub Issues + Pull Requests（用户已选择）。
- 社区规范文件：最小集（README + LICENSE(MIT) + CONTRIBUTING）（用户已选择）。
- 项目命名：中文“游牧岛”，后端对外标题为 “Nomad Island API”（来自 ymd-server/app/core/config.py）。

## Proposed Changes（落地改动）
### 1) 新增根目录 README.md（项目介绍）
- 文件：/README.md
- 内容结构（建议固定为以下段落，便于读者快速扫读）：
  - 一句话定位：游牧岛是什么、解决什么问题（共居生活方式/活动组织/社区内容沉淀）。
  - 功能概览（与当前代码一致）：
    - 用户与认证：邮箱登录/注册、JWT（可提及小程序场景但不承诺已上架）。
    - 活动：列表/详情/报名/我的活动。
    - 社区：发帖/详情、媒体（图片/音频组件）、互动（如已存在再写：点赞/收藏/分享等）。
    - 共居：空间列表/详情、咨询/登记（inquiries）。
    - 积分：积分中心、积分流水（如已存在接口则写）。
    - 管理后台：用户/帖子/评论/活动/咨询/积分配置与调整（以 admin 现有页面为准）。
  - 技术栈与架构（高层即可）：
    - ymd-app：UniApp(Vue3) + Pinia + Vite
    - ymd-admin：Vue3 + Vite + Pinia
    - ymd-server：FastAPI + PostgreSQL + SQLAlchemy(Async) + Alembic
    - 部署：docker-compose（api + web；uploads volume）
  - 快速开始（面向贡献者的最短路径）：
    - 使用根目录 dev-start.ps1（动态端口）启动三端，并引用 scripts/README-dev-start.md 的说明。
    - 说明环境变量位置：各端的 .env.example（避免泄露真实密钥）。
  - 目录结构速览（只列顶层与三端即可）。
  - Roadmap（轻量、可共建）：用 5–8 条可讨论的方向，作为 Issues 的起点（不做时间承诺）。
  - Contributing：一句话引导到 CONTRIBUTING.md，并说明默认流程：Issue → PR → Review。
  - License：明确 MIT，并链接到 LICENSE。

### 2) 新增根目录 LICENSE（MIT）
- 文件：/LICENSE
- 内容：标准 MIT License 正文。
- 版权行：
  - 默认格式：Copyright (c) 2026 <YOUR_NAME>
  - 在落地执行时，用仓库所有者/组织名替换占位符（如果仓库存在 package.json author 或已有统一署名则以其为准）。

### 3) 新增根目录 CONTRIBUTING.md（最小共建指南）
- 文件：/CONTRIBUTING.md
- 内容要点（保持“最小可执行”）：
  - 参与方式：报 Bug / 提需求 / 提 PR（都以 Issue 为入口更可追踪）。
  - 开发前准备：指向 scripts/README-dev-start.md；强调先跑起来再改。
  - 分支与提交：建议约定（如 feature/xxx、fix/xxx），以及 PR 描述需包含“动机/改动/验证方式”。
  - 代码风格：不强行引入新规范；优先遵循各子项目现有格式与 lint（如果已有）。
  - 验证清单：最少要求（能启动、关键路径自测、必要时补测试）。
  - 安全提示：不要提交真实密钥/账号；使用 .env.example。
