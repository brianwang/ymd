# 社区模块页面设计（Desktop-first）

## 全局（适用于社区模块所有页面）
### Layout
- 采用“顶栏 + 主内容容器”结构；主容器最大宽度 960px，左右居中。
- 信息流与列表使用 Flexbox + 卡片栅格（桌面单列为主，宽屏可切换双列）；移动端降级为单列。

### Meta Information
- 社区首页：title「社区 - 游牧岛」；description「浏览最新动态，关注同路人」。
- 帖子详情：title「帖子详情 - 游牧岛」；description「查看帖子内容与点赞状态」。
- 发帖页：title「发布动态 - 游牧岛」。
- 用户主页：title「用户主页 - 游牧岛」。

### Global Styles
- 背景：#F7F7F8；卡片背景：#FFFFFF；分割线：#EDEDED。
- 主色：#111111（文本）；强调色：#2563EB（主按钮/链接）；危险色：#DC2626（删帖）。
- 字体层级：H1 24/32，H2 18/28，正文 14/22，辅助 12/18。
- 按钮：主按钮实心；次按钮描边；Hover（桌面）：提升阴影/背景加深 4%。
- 点赞态：未点赞为灰色描边图标；已点赞为强调色填充。

## 1）社区首页（信息流）
### Page Structure
- 顶栏（固定）：左侧「社区」，右侧「发布」按钮。
- 主内容：信息流列表（卡片）。

### Sections & Components
1. 顶栏（Top Bar）
   - 标题：社区
   - 主操作：按钮「发布」→ 跳转发帖页
2. 信息流卡片（Post Card）
   - 头部：作者头像 + 昵称（可点）+ 时间
   - 正文预览：最多 3 行折叠
   - 图片预览：1-3 张宫格（点击进入详情页）
   - 互动区：点赞按钮（含计数）
   - 交互：
     - 点击作者 → 用户主页
     - 点击正文/图片 → 帖子详情
     - 点击点赞 → 立即切换状态；失败提示并回滚
3. 列表状态
   - 首次加载 Skeleton
   - 空态：提示「还没有动态」
   - 加载更多：底部加载条

### Responsive
- >= 1024px：卡片容器 960px；卡片间距 16px。
- < 768px：顶栏高度缩小；卡片全宽；图片宫格间距 4-6px。

## 2）帖子详情页
### Page Structure
- 顶栏：返回、标题「帖子详情」、（若有权限）更多菜单。
- 主内容：帖子卡片（完整版）。

### Sections & Components
1. 帖子主体（Full Post）
   - 作者区：头像/昵称（可点）+ 发布时间
   - 正文：完整展示
   - 图片：可左右滑动/点击放大预览
2. 互动区
   - 点赞按钮 + 点赞数
3. 权限操作（仅作者/管理员可见）
   - 更多菜单：选项「删除」
   - 删除二次确认弹窗：
     - 文案「删除后不可恢复，确定删除？」
     - 按钮：取消 / 删除（危险色）
   - 删除成功：Toast 提示并返回社区首页

---

## 3）发帖页
### Page Structure
- 顶栏：返回、标题「发布动态」、右侧「发布」主按钮（禁用态支持）。
- 主内容：正文输入 + 图片选择区。

### Sections & Components
1. 正文输入
   - 多行 Textarea：placeholder「分享你的动态…」
   - 字数提示（可选）：右下角显示当前字数
2. 图片选择
   - 上传入口：加号卡片
   - 已选图片：缩略图列表（可删除单张）
3. 发布按钮逻辑
   - 禁用态：正文为空且未选择图片
   - 点击发布：展示加载态；成功后返回社区首页
4. 异常提示
   - 网络失败/校验失败：Toast 文案明确（如「请填写文字或至少上传一张图片」）

---

## 4）用户主页（社区维度）
### Page Structure
- 顶栏：返回、标题（用户昵称）。
- 主内容：用户信息卡 + 用户帖子列表。

### Sections & Components
1. 用户信息卡（Profile Header）
   - 头像、昵称、简介（如有）
   - 关注按钮：
     - 未关注：主按钮「关注」
     - 已关注：次按钮「已关注」点击后变更为「取关」（二次确认可选）
     - 若查看自己：隐藏关注按钮
2. 用户帖子列表
   - 列表项与社区首页卡片一致（可复用组件）
   - 点击进入帖子详情
3. 列表状态
   - 空态：提示「Ta 还没有发布内容」

---

## 关键交互与状态（跨页面一致）
- 未登录用户点击「关注/点赞/发布/删除」：统一弹出登录引导（模态或页面跳转）。
- 幂等交互：重复点击关注/点赞不产生重复记录；UI 以服务端最终结果为准。
- 软删帖子：被删除帖子在信息流与用户主页列表中不再展示（刷新后消失）。
